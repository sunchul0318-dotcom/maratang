# 제품 요구사항 명세서 (PRD) - 팀 랜덤 선택기

## 1. 개요 (Overview)
- **프로젝트 명**: 팀 랜덤 선택기 (Team Random Selector)
- **목적**: 팀 내 회식 메뉴, 발표 순서, 역할 분담 등을 빠르고 재미있게 랜덤으로 결정하는 웹 애플리케이션 제공.
- **핵심 가치**: 사용자 친화적인 직관적 UI, 매끄러운 애니메이션을 통한 뛰어난 UX, 단순한 공유 방식 제공.

## 2. 사용 대상 (Target Audience)
- 모임, 회사, 팀 단위에서 무언가를 랜덤하게 정해야 하는 누구나.

## 3. 핵심 기능 (Key Features)
### 3.1. 항목 입력 기능
- **텍스트 직접 입력**: 사용자가 줄바꿈을 기준으로 항목(이름, 메뉴 등)을 입력할 수 있는 필드 지원.
- **파일 업로드**: `.txt` 파일을 업로드하여 리스트를 한 번에 불러오는 기능.
- **URL 파라미터 파싱 및 공유**:
  - `도메인/?names=A,B,C` 와 같이 접속했을 때 파라미터를 읽어 입력창에 A, B, C 항목이 자동으로 설정되어야 함.
  - 항목 입력 후 생성된 '공유 링크'를 팀원들에게 쉽게 복사하여 전달 가능.

### 3.2. 랜덤 추첨 기능
- **단일 선택**: 목룍 중에서 결과로 1개 선택 (예: 오늘 발표자 1명 지정).
- **순서 섞기 (셔플)**: 목록 전체의 순위를 랜덤으로 지정 (예: 팀 발표 순서 정하기).
- **애니메이션 효과**: 결과가 나오기 전 시각적인 긴장감을 유도할 수 있는 모던한 애니메이션 (예: 룰렛, 슬롯머신, 카드 뒤집기 등).

### 3.3. 결과 확인
- 명확하고 돋보이는 결과 디스플레이.
- 다시 뽑기 (재추첨) 기능 지원.

## 4. UI/UX 및 디자인 요구사항
- **디자인 컨셉**: Modern & Simple. 트렌디한 색상 팔레트와 글래스모피즘(Glassmorphism), 부드러운 그라데이션 등을 활용해 심심하지 않은 고급스러운 퀄리티 확보.
- **반응형 웹**: 데스크탑 뿐만 아니라 모바일 브라우저에서도 시각적 깨짐이 없어야 함.
- **애니메이션**: UI/UX 상승을 위해 화면 전환, 호버 효과, 추첨 액션에 마이크로 인터랙션을 반영.

## 5. 기술 스택 (Tech Stack)
- **프론트엔드 프레임워크**: Next.js (App Router / React)
- **스타일링**: Tailwind CSS
- **애니메이션**: CSS Transitions & 애니메이션 라이브러리 (예: Framer Motion)
- **인프라/배포**: Cloudflare Pages
